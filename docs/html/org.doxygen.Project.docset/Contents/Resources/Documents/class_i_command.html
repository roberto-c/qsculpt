Detailed Description

Command interface. Every application command should implement this interface.

The command can have several configuration parameters.

virtual void ICommand::activate ( bool  active) [pure virtual]

Set the command as the active one. This is called with the parameter in true to indicate that the command is active and should be prepared to accept the input from the user (like mouse events, etc).

To set the command as a non active, then pass false in the active parameter.

Parameters:
activeTrue, the command is active. False, the command is no longer active.

Implemented in TransformCommand, CommandBase, BrushCommand, and TransformCameraCommand.

virtual ICommand* ICommand::clone ( ) const [pure virtual]

Creates a new instance of the command. The new command instance should have, at least, the same configuration parameters as the original one.

virtual void ICommand::execute ( ) [pure virtual, slot]

Execute the command in no interactive way. This means, that once that all the parameters of the command are set, the command will be executed without user intervention.

virtual bool ICommand::needsUserInteraction ( ) const [pure virtual]

States if the command needs the user interaction for its execution.

This should return TRUE if the command depends on user input like mouse movement. This function should return false if the command does not require user intervention (its action is done immediatly).
